What Is This?

SOC Lab is a self-contained environment for learning security monitoring, log analysis, detection engineering, incident response, threat intelligence, SOAR automation, and controlled attack simulation.

Detect

Wazuh, Elastic, Kibana, Logstash, Filebeat, and Metricbeat collect and analyze security events.

Respond

TheHive, Cortex, and Shuffle help turn alerts into cases, enrichment, and automation workflows.

Practice

DVWA and Juice Shop provide safe local targets for testing alerts and dashboards.

Quick Start

Windows Docker Desktop

powershell -ExecutionPolicy Bypass -File .\scripts\setup.ps1
docker compose ps

Linux / Kali

chmod +x scripts/*.sh
sudo sysctl -w vm.max_map_count=262144
./scripts/setup.sh
docker compose ps

Linux / Kali With Suricata and Zeek

WITH_LINUX_NSM=1 ./scripts/setup.sh

Suricata and Zeek use host networking and packet capture. Keep them disabled on Windows Docker Desktop.

Architecture

  1. Agents, test targets, and Docker logs generate telemetry.
  2. Wazuh and Elastic ingest, parse, index, and alert on events.
  3. Wazuh custom integrations can send high-severity alerts to TheHive and MISP.
  4. TheHive manages investigations while Cortex enriches observables.
  5. Shuffle can automate response actions through workflows.
  6. Grafana and Prometheus provide operational visibility.

Troubleshooting

Docker Not Running

docker info

Start Docker Desktop on Windows or the Docker service on Linux/Kali.

Linux Kernel Setting

sudo sysctl -w vm.max_map_count=262144

Elasticsearch and Wazuh indexer need this setting on Linux hosts.

Ports in Use

docker compose ps
netstat -ano | findstr :443

Change ports in .env if another local service is already using them.

Clean Reset

docker compose down -v
docker compose pull
docker compose up -d

This deletes lab data and starts fresh.
